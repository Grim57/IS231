## Содержание курсовой работы (более детально и актуально)  

### Введение  
Актуальность (онлайн торговли для компании из вашей отрасли)  
Цель - основная из темы  
Задачи - перечень соответствует подразделам основной части (содержанию)  
Объект - процесс онлайн продаж, интернет торговли из отрасли по теме  
Предмет - интернет магазин, веб-сервис, динамический веб-сайт (взять из темы)    

### Основная часть  
1. Теоретическая часть  
1.1 Анализ предметной области   
1.2 Сбор и анализ требований  
   1.2.1 Составление анкеты с вопросами для заказчика (брифа)  
   1.2.2 Разработка технического задания по брифу с ответами заказчика  
1.3 Архитектурное проектирование  
(Выбранная модель разработки - итеративная. Спецификация требований для 1 итерации. Архитектура проекта - клиент-серверная. Выбранная СУБД.
Паттерн реализации MVC (Model-View-Controller))  
1.4 Детальное проектирование с использованием UML диаграмм  
(диаграмма компонентов системы, диаграммма последовательности для оформления заказа, диаграмма классов проекта, ER диаграмма для сущностей БД и связей между ними)
+ тестовые сценарии, тестовые кейсы  
+ оценку нефункциональных требований с помощью метрик    
+ оценка качества кода по метрикам (сложности, сопровождаемости - phpMetrics)  

2. Практическая часть
2.1 Разработка ИС
- выбор и характиристики IDE - Visual Studio Code (сравнение с другими, преимущества)  
- использование фреймворка Bootstrap для интерфейсной, клиентской части, его преимущества  
- создание модульных тестов (валидация данных формы, скриншот)  
- реализация основного функционала - путь клиента: регистрация, авторизация, выбор товара, каталог, оформление заказа (подробно, в скриншотах работающей программы)

включает следующие разделы:  

1) Введение:  
o Цель проекта (что вы хотите достичь).  
o Общая концепция сайта.  

2) Требования:  
o (2.1) Функциональные:  
Формулируются глаголами: "система должна предоставлять", "должна обеспечивать"  
Должны быть конкретными и проверяемыми  
Пример:  
"1.2. Система должна предоставлять возможность:  
- Выбора пиццы из каталога  
- Индивидуального составления пиццы  
- Оформления заказа в 3 шага  

o (2.2) Нефункциональные:    
- Производительность (время отклика, нагрузка)  
- Безопасность (защита данных, авторизация)  
- Надежность (время безотказной работы, нагрузка, восстановление)  
- Юзабилити (удобство интерфейса)  
Пример:  
"3.1. Время загрузки страниц не должно превышать 2 секунд при скорости интернета 10 Мбит/с"
